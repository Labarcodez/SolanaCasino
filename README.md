# OrbitCasino — Real SOL Gambling on Solana

A production-ready Solana casino where users gamble **real mainnet SOL**. Built in the style of [solpump.io](https://solpump.io) and Rugs.fun.

**Brand:** OrbitCasino — customize name, logo, and copy in `frontend/src/lib/brand.ts`.

## Features

- **Real SOL gambling** on Solana mainnet — no mock balances
- **Phantom wallet** sign-in (browser extension, no app ID required)
- **Email sign-in** via Google/Apple OAuth (Phantom embedded wallet, requires Portal app ID)
- **Wallet signature authentication** — JWT sessions, protected API & WebSocket
- **Crash game** — live multiplier, auto-cashout, provably fair
- **Coinflip** — instant heads/tails, 95% RTP
- **On-chain Anchor program** — vault PDA, player accounts, provably fair crash & coinflip
- **Hybrid architecture** — on-chain balances + off-chain WebSocket oracle for crash UI
- **Provably fair verification** — UI + API to verify crash rounds
- **Docker deployment** — single-container production setup

## On-Chain Architecture (Anchor)

This project follows the [Solana game development guide](https://solana.com/developers/guides/games/getting-started-with-game-development) hybrid pattern:

| Layer | Responsibility |
|-------|----------------|
| **Anchor program** (`programs/solcasino`) | Vault PDA, player balances, bets, commit-reveal fairness |
| **Backend oracle** | Signs `start_round`, `start_running`, `finalize_round`; WebSocket multiplier ticks |
| **Frontend** | Phantom wallet signs `deposit`, `withdraw`, `place_bet`, `cashout`, `coinflip_bet` |

### Phantom Portal Setup (Email Login)

1. Register at [phantom.com/portal](https://phantom.com/portal)
2. Create app → copy App ID → set in `frontend/.env` and `backend/.env`
3. Allowlist URLs:
   - `http://localhost:5173` (dev)
   - `http://localhost:5173/auth/callback` (dev)
   - Your production domain + `/auth/callback`

**Phantom browser extension works without an app ID.**

## How It Works

### Authentication
1. User connects Phantom wallet (extension or email)
2. User signs a free authentication message (no SOL spent)
3. Backend issues a 7-day JWT session
4. All bets, deposits, withdrawals require valid session

### Deposits
1. User clicks **Deposit SOL** → signs real `SystemProgram.transfer` to casino wallet
2. Backend verifies transaction on-chain (with retry/polling)
3. Casino balance credited

### Playing
- **Crash**: Bet during 8s window → cash out before crash (or auto-cashout)
- **Coinflip**: Pick heads/tails → instant result

### Withdrawals
- **With private key**: Instant SOL transfer from casino wallet
- **Without private key**: Queued withdrawal (balance deducted, processed manually)

### Provably Fair
Each crash round publishes `serverSeedHash` before betting. After crash, `serverSeed` is revealed. Verify in the **Fairness** tab or via API.

| Event | Direction | Description |
