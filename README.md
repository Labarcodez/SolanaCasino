## On-Chain Architecture (Anchor)

This project follows the [Solana game development guide](https://solana.com/developers/guides/games/getting-started-with-game-development) hybrid pattern:

| Layer | Responsibility |
|-------|----------------|
| **Anchor program** (`programs/solcasino`) | Vault PDA, player balances, bets, commit-reveal fairness |
| **Backend oracle** | Signs `start_round`, `start_running`, `finalize_round`; WebSocket multiplier ticks |
| **Frontend** | Phantom wallet signs `deposit`, `withdraw`, `place_bet`, `cashout`, `coinflip_bet` |

## How It Works

### Authentication
1. User connects Phantom wallet (extension or email)
2. User signs a free authentication message (no SOL spent)
3. Backend issues a 7-day JWT session
4. All bets, deposits, withdrawals require valid session

### Deposits
1. User clicks **Deposit SOL** → signs real `SystemProgram.transfer` to casino wallet
2. Backend verifies transaction on-chain (with retry/polling)
3. Casino balance credited

### Playing
- **Crash**: Bet during 8s window → cash out before crash (or auto-cashout)
- **Coinflip**: Pick heads/tails → instant result

### Withdrawals
- **With private key**: Instant SOL transfer from casino wallet
- **Without private key**: Queued withdrawal (balance deducted, processed manually)

### Provably Fair
Each crash round publishes `serverSeedHash` before betting. After crash, `serverSeed` is revealed. Verify in the **Fairness** tab or via API.
